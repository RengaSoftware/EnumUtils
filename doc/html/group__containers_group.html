◆ variadic_type

using variadic_type = decltype(detail::make_variadic_type<TContainer, E, TItem>(sequence_type<E>{}))

A type that denotes an arbitrary variadic container which can contain a set of types mapped to each item of E.

The above description is rather cryptic. The following example will try to elaborate on that a little:

#include <enum_utils/mapping.h>
#include <enum_utils/traits.h>
#include <enum_utils/variadic.h>
#include <tuple>
#include <type_traits>
enum class type { integer, real, _last = real };
ENUM_UTILS_DEFINE_TRAITS(type, _last)
ENUM_UTILS_DEFINE_MAPPING_TO_TYPE(type::integer, int)
ENUM_UTILS_DEFINE_MAPPING_TO_TYPE(type::real, double)
template <type t>
using mapped_type = enum_utils::mapped_type_t<type, t>;
using mapped_type_tuple = enum_utils::variadic_type<std::tuple, type, mapped_type>;
static_assert(std::is_same<mapped_type_tuple, std::tuple<int, double>>::value, "OK");

This becomes especially powerful when used on std::variant or boost::variant. You can, of cousre, make your own variadic containers. They'd work here too.
